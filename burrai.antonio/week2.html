Computer-Aided Design

.

The weekly assignment focuses on 2D and 3D modeling. My final project will be a fish, so a three-dimensional shape, that I'd like to model using free hand tecniques (in this stage of design). On advanced stages of design, more complexity and geometrical precision will be needed, so mathematical 3D modeling will be used. Please NOTE that free hand tecniques have been used, so at this level of design there are not 2D files. A complete 2D analisys has been developed in the final project.

I do not really want to use an existing project, and I'd really like to improve a new skill, so I started using Autodesk Meshmixer, that is very similar to ZBrush. Modeling a fish is very complex, especially by free hand. At the beginning I was thinking in a carp, but at the end my model looks like a fat shark.

It has been a big problem the high number of triangular element derived from a free hand modeling. I need to spend some time in order to find the best way to reduce the triangles. Several reduction possibilities are available in Meshmixer.

Another big problem, not immediately visible but that affect the 3d print, was the presence of "bubbles" resulting from the "INFLATE" brush.

The mentioned "bubbles" were not visible in CURA , even analyzing the layers. I've noted that layer number 150, almost at the end of the caudal fin, it wasn't the last one ! More that 400 layers have been defined by Cura.

I've tried to print the 3d model of Piske using a Falla3D printer . The error in g-code causes an apparent illogical movement of the extrusion system at the beginning of the caudal fin. After a few test it was not possible to proceed without adjusting the model. The final 3d model is not so easily changeable (it's composed only of small triangular meshes, so I switch to a mathematical 3D model. For a preview of the final 3D model, please refer to the Final Project page.
